## 🔴 Critical Issues (Phải sửa ngay)

### 1. API Keys bị lộ trong file `.env`
- **File:** `.env` (dòng 2, 13)
- **Nguy hiểm:** File `.env` chứa **Telegram Bot Token** và **Gemini API Key** dạng plain text. Mặc dù `.gitignore` đã exclude `.env`, nhưng nếu lỡ push lên public repo hoặc share source code, hacker sẽ **chiếm toàn quyền bot** Telegram và **dùng hết quota API** Gemini của anh.
- **Token bị lộ:**
  - `TELEGRAM_BOT_TOKEN=8377561490:AAGPN8I_d_TS...` 
  - `GEMINI_API_KEY=AIzaSyAhsTeOVJag...`
- **Cách sửa:**
  1. **Đổi ngay** cả 2 token/key (vào BotFather tạo token mới, vào Google AI Studio tạo key mới)
  2. Đảm bảo `.env` luôn nằm trong `.gitignore` ✅ (đã có)
  3. Không bao giờ commit `.env` vào git

### 2. 9 chỗ dùng `except:` (bare except) - Nuốt lỗi im lặng
- **Files:** `monitor.py` (5 chỗ), `scraper_full.py`, `search_handler.py`, `recover_data.py`
- **Nguy hiểm:** Khi có lỗi, code nuốt im lặng và tiếp tục chạy. Ví dụ nếu database bị corrupt, bot vẫn chạy bình thường nhưng **trả kết quả sai** mà anh không biết.
- **Cách sửa:** Đổi `except:` → `except Exception as e:` và log lỗi ra. Ví dụ:
  ```python
  # ❌ Hiện tại
  except:
      pass
  
  # ✅ Sửa thành
  except Exception as e:
      logging.warning(f"Lỗi khi parse: {e}")
  ```

- **File:** `monitor.py` (`_detail_cache`, `_report_cache`)
- **Vấn đề:** Mỗi lần user search, kết quả được cache mãi mãi không xóa. Bot chạy lâu → RAM tăng dần.
- **Cách sửa:** Thêm TTL (time-to-live) hoặc giới hạn cache size (ví dụ max 100 entries, xóa cũ nhất)

- **Vấn đề:** Project nặng không cần thiết, khó phân biệt file production vs debug.
- **Cách sửa:**
